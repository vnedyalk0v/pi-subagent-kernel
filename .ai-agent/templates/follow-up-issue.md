# Follow-up issue template

Use when a valid finding is real but out of scope for the active issue/PR.

```markdown
## Goal
<what should be fixed or improved>

## Source
Found while working on #<issue-number> / PR #<pr-number>.

## Evidence
- <file/path/test/source evidence>

## Acceptance criteria
- [ ] <criterion 1>
- [ ] <criterion 2>

## Non-goals
- <what this issue should not cover>

## Suggested labels
- priority:<p0/p1/p2/p3>
- type:<feature/bug/test/chore/security/docs/research>
- area:<area>
- phase:<mvp/alpha/beta/post-mvp>

## Suggested milestone
<milestone>
```
